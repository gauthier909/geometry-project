1 Introduction
We have chosen to base our project on one of the articles proposed on the forum[8]. The article is
about Isoperimetric Enclosures. The problem is as follows: Let S be a set of n >2 points in the plane
and let P >0 be a given value. Among all the curves of perimeter P that enclose S, which is the curve
that encloses the maximum area ?

2.1 Isoperimetric theorem
The isoperimetric theorem says that of all convex n-gons of a given perimeter, the one which maximizes
area is the regular n-gon.[4]
If we accept all n , we have that the convex n-gon that maximize the area is the innite-gon which
is the circle.
On the webpage of this section we can compare equilateral triangles, squares and circles with a
xed perimeter and see that the circle cover a bigger area than the equilateral triangle or the square.
2.2 Convexity of a xed perimeter curve of maximum area
Considering a closed curve with a xed perimeter that maximize the area inside the curve. This curve
must be convex or else, it is not the curve that maximize the area for the given perimeter.
We will intiutively show this by the fact that, if we take two points of the curve for which the
segment joining these two points is outside the curve (which can be the case with a non-convex curve),
we can do a reection of the part of the curve between our two points with the segment joining these
two points. With this reection we obtain a new curve with same perimeter but the area that the
curve covers is bigger.
On the webpage of this section we show an example to this with a non-convex polygon that we
transform following the method described above to obtain a same perimeter convex polygon but that
covers a bigger area.
2.3 Circular arc
[3] Lets consider an open curve joining two endpoints of a segment to form a closed curve of maximum
area for a xed perimeter. Then this open curve is a circular arc. We can intiutively see it if we extend
this open curve to a full circle. We would obtain a circle and the two endpoints of the segment would
be points of the circle. If we change the shape of the open curve while maintaining the same perimeter,
we would no more have a circle. This go against the isoperimetric theorem, and thus, this new curve
is not of maximum area.

Now considering our original problem : If we want to enclose a convex polygon inside a closed
curve of maximum area with a xed perimeter, we would intuitively obtain a sequence of one or more
curves, each one being either a circular arc of similar radius or a segment ush of the polygon.
The reason why the circular arcs should be of similar radius is because a closed curve of maximum
area has to be convex (see Convexity). For the same reason, the endpoints of these circular arcs have
to be vertices of the polygon.
To show that they must be of similar radius, we will consider a simple polygon (equilateral triangle)
thats enclosed by a sequence of circular arcs of dierent radii and we will then swap a part of each
circular arc (the segments connecting the two endpoints of each part of circular arc are of same length)
with the part of one of the other two circular arc. We will see that we will obtain a non-convex curve
of same perimeter than the one before the swapping. And as we have seen, a closed curve of maximum
area must be convex and thus the circular arcs must be of same radius.
To see this, try on the canva below to put 85 for radius 1, 180 for radius 2, and 130 for radius 3.

2.4 Voronoi diagram
The basic voronoi diagram consist in nitely many points (seeds) in a plane. For each points there is
a corresponding region, called a Voronoi cell, consisting of all points of the plane closer to that seed
than to any other[5].
But in order to compute the enclosed curve of maximum area with a predetermined perimeter, we
will need the farthest-point Voronoi diagram. It is a variant of the Voronoi diagram, in which each
cell of the FPVD (farthest-point Voronoi diagram) contains all the points of the plane that has for
farthest seed, the same one. So for each cell, instead of containing all the points that share the same
closest seed , they share the same farthest seed.
The FPVD can be seen as a tree with n unbounded edges and we can think of the leaves of this
tree as points at innity in the direction of these unbounded edges[4]
In order to construct the farthest point voronoi diagram[2] we rst need to construct the convex
hull[1] of the points (seeds) to determine the farthest points because a point that is not part of the
convex hull cannot be a farthest point of a region because there will always be another one that is
farther and that is part of the convex hull. And then we will construct the smallest enclosing circle
(using Welzls algorithm[7][6]) because of some properties. Notably the fact that the center of the
smallest enclosing circle lies either on a vertex or an edge of the FPVD.
On the webpage of this section there is a canva to compute the farthest point voronoi diagram from
a set of point. We can create this set of points by clicking on Add random point button (create a
point at random coordinates). Then we compute the convex hull by clicking on Draw CH button.
Then we compute the smallest enclosing circle by clicking on Smallest circle button. And nally we
compute the FPVD by clicking on FPVD button. There is also Clear button to clear everything.
2.5 Isoperimetric Enclosures
Let S be a set of n >2 points in the plane whose convex hull has perimeter t. Given a number P >=
t, we have the following problem: Of all curves of perimeter P that enclose S, which is the curve that
encloses the maximum area ?
To answer this question, we will use the following algorithm from the article:
1. Compute the smallest enclosing circle that contains S (and thus its convex hull) and compare
the perimeter of this circle with P.
2. If P >= perimeter of the smallest enclosing circle : construct the circle that has the same center
than the smallest enclosing circle with perimeter P.
3. If P <perimeter of the smallest enclosing circle : As we have seen, the solution will look like a
sequence of circular arcs with the same radius. First compute the FPVD of S and register all
the vertices v of the FPVD with their p(v). With p(v) being the radius of the smallest enclosing
circle with center v. Then sort the vertices by their value under p. We can then approximate
the radius of the arcs by doing a binary search in the set p(v), v is a vertex of the FPVD of S.
We will nd vertices v and u, such that for the correct radius r, we have p(v) <r<p(u). At

this point, the curve we are looking for has the same combinatorial structure for any radius p(v)
<r <p(u). We have thus a combinatorial description of the curve where only the radius p(v)
<r<p(u) has yet to be determined.
